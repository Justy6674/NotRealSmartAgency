# Preferences

- Be direct, no filler
- Australian English always
- Don't summarise what was just done — user can read the diff
- When frustrated, stop patching and plan properly
- PLAN AND ASK before building anything complex

## Development
- ALWAYS test locally with `npm run dev` before pushing
- Use Playwright headed mode to screenshot and verify visually
- Read installed skills BEFORE writing code that relates to them
- Never push untested 3D/WebGL code to production
- If unsure, revert to Coming Soon rather than push broken
- Use `force-dynamic` on all pages with base-ui components
- IBM Plex font, silver/chrome metallic palette (oklch hue ~240)
